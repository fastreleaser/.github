# Contributing

Thanks for helping improve fastreleaser.

## Before you start

- Open an issue for bugs, gaps, or feature ideas when the change needs discussion.
- Keep changes focused and easy to review.
- Prefer small pull requests over bundled refactors.

## Development

1. Fork or branch from the target repository.
2. Make the smallest change that solves the problem.
3. Add or update tests when behavior changes.
4. Update docs when the user-facing workflow changes.

## Pull requests

- Explain the problem and the chosen fix.
- Call out behavior changes, migrations, or follow-up work.
- Include test evidence for non-trivial changes.

## Review expectations

- Be direct, kind, and specific.
- Treat review comments as collaboration, not ceremony.
- Resolve feedback with follow-up commits unless the repository prefers squashing before merge.
